      # 开发规范

      # 1.uniapp规范

      • 页面文件遵循 Vue 单文件组件 (SFC) 规范
      • 组件标签靠近小程序规范，详见uni-app 组件规范
      • 接口能力（JS API）靠近微信小程序规范，但需将前缀 wx 替换为 uni，详见uni-app接口规范
      • 数据绑定及事件处理同 Vue.js 规范，同时补充了App及页面的生命周期
      • 为兼容多端运行，建议使用flex布局进行开发

      # 2.命名规范

      类型 命名规则
      构造函数 大驼峰
      方法命名 小驼峰
      变量名 小驼峰
      组件名 大驼峰
      样式类 小写，中划线连接
      方法内变量 下划线开头，小写

      # 3.注释规范

      • 文件第一行注释文件的基本信息
      • 声明方法需注释方法用途
      • 组件内HTML大代码块用注释分隔

      # 多语言配置

      # 1.lang 文件夹下面写国际化语言的逻辑

      1.将所需要的文件引入（en.js zh.js vue-i18n 等） 2. 获取设备信息 ，并保存本地 目的: 是为了知道用户手机用的是什么语言，方便用户一进来看到的就是他设备设置的语言，我这里是将获取到的信息全部保存了（项目中需要），也可以只保存system_info.language 代码如下（写在lang/index.js文件中）：

      import LangEn from './en.js'
import LangChs from './zh.js'
import Vue from 'vue'
import VueI18n from './vue-i18n'
Vue.use(VueI18n)
const system_info = uni.getStorageSync('system_info')
if (!system_info) {
	// 获取设备信息
	uni.getSystemInfo({
		success: function (res) {
			uni.setStorageSync('system_info', res);
		}
	})
}
	const cur_lang = system_info.language == 'en' ? 'en' : 'zh_CN'
	const i18n = new VueI18n({
		locale: cur_lang || 'zh_CN',  // 默认选择的语言
		messages: {  
				'en': LangEn,
				'zh_CN': LangChs
			}
	})
	export default i18n

      # 3.项目中引入

      uniapp 不支持在取值表达式中直接调方法，因此，$t方法不可用，所以通过计算属性的方式：

      <template>
	<view class="content">
		<image class="logo" src="/static/logo.png"></image>
		<view class="text-area">
			<text>{{ i18n.game }}</text>  
		</view>
		<button type="primary" @tap="change">切换语言</button>
	</view>
</template>
<script>
export default{
	computed: {  
 i18n () {  
    return this.$t('index')  
 }  
}
}
</script>
